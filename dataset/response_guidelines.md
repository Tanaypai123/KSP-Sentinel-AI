# Police Intelligence UX Guidelines: Response Format

This document details the standardized response guidelines for **KSP Sentinel AI** outputs.

---

## 📋 Design Philosophy

Sentinel AI responses are designed for **law enforcement officers operating under operational pressure**. The layout prioritizes:
1. **High Visual Scannability**: Predictable section headers, clean spacing, and distinct emojis.
2. **Minimalist Formatting**: No heavy markdown clutter (such as complex tables, nested asterisks, or dense blocks) to guarantee legibility on small tactical displays.
3. **Evidence Transparency**: Every claim made in the summary is explicitly linked to its database source and confidence metric.

---

## 🔍 Standard Layout Sections

Every Sentinel AI response summary must consist of exactly 8 standardized text sections, formatted with clean uppercase headers and dashed underlines:

### 1. 📋 EXECUTIVE SUMMARY
* **Purpose**: A single-line overview of the retrieved intelligence.
* **Format**: Plain text statement detailing retrieved counts or target objects.
* **Example**: `I retrieved the details for FIR KSP-2024-0001.`

### 2. 🔍 KEY FINDINGS
* **Purpose**: Key details extracted from the database records.
* **Format**: A clean bulleted list where fields are separated by horizontal pipes (`|`). No tables or markdown tables.
* **Example**:
  ```
  • FIR: KSP-0001 | Crime: Robbery | District: Bengaluru Urban | Status: Closed
  ```

### 3. 🗃️ EVIDENCE
* **Purpose**: Source table names, query constraints, and matching record counts.
* **Format**: Clean source summary with record sizes.
* **Example**:
  ```
  • Source: database: case_master table filtered by crime_no='KSP-0001' (2 records).
  ```

### 4. 📊 ANALYTICS
* **Purpose**: Macroscopic crime analysis (Crime patterns, hotspots, associate networks).
* **Format**: Multi-line summaries generated by AI sub-engines.
* **Example**:
  ```
  • Crime Pattern: High frequency of property theft observed during holiday weekends.
  • Hotspots: Risk Zones: Sector A | Peak Hours: 22:00-02:00
  ```

### 5. 🧠 REASONING
* **Purpose**: The chain of evidence validating the response claims.
* **Format**: Clean logic statements with a final verification conclusion.
* **Example**:
  ```
  • The requested FIR details were successfully retrieved from the database.
  • Conclusion: Conclusions strictly supported by verified data.
  ```

### 6. 💡 RECOMMENDATIONS
* **Purpose**: Actionable next steps for investigators.
* **Format**: Specific task list with prioritized actions.
* **Example**:
  ```
  • Vehicle Verification (Priority: MEDIUM) - Verify RTO records for vehicles registered to suspects.
  ```

### 7. 🎯 CONFIDENCE
* **Purpose**: Total confidence percentage and risk assessment.
* **Format**: Calculated confidence metric followed by base classification and penalization factors.
* **Example**:
  ```
  • Score: 63.0%
  • Risk Level: MEDIUM
    - Base confidence initialized to 0.53 from intent classifier.
    - Reasoning engine applied positive adjustment (+0.10).
  ```

### 8. ⚠️ WARNINGS
* **Purpose**: Edge-case warnings, low confidence alerts, or filter limits.
* **Format**: Specific warnings list or safe status confirmation.
* **Example**:
  ```
  None - Safe execution under current bounds.
  ```
